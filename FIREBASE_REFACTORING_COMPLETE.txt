## Technical Details

### Firestore Data Structure
```
Collections:
├── sessions/
│   └── [sessionId]
│       ├── sessionId: string
│       ├── type: string
│       ├── startTime: string (ISO)
│       └── active: boolean
│
└── attendance/
    └── [teamId_sessionId]
        ├── teamId: string
        ├── sessionId: string
        ├── sessionType: string
        ├── timestamp: string (ISO)
        ├── imageURL: string
        └── status: string
```

### Firebase Storage Structure
```
gs://bucket-name/
└── selfies/
    ├── TEAM001_session1_1234567890.jpg
    ├── TEAM002_session1_1234567891.jpg
    └── ...
```

### Image Processing Flow
```
Base64 Image
    ↓ (decode)
Image Bytes
    ↓ (PIL compress: 1920x1440, 90% JPEG)
Compressed JPEG (~200KB)
    ↓ (upload to Storage)
Public URL
    ↓ (store in Firestore)
Attendance Record with imageURL
```
